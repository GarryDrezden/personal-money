# 01. Обзор продукта

## Что это

**«Личный бюджет»** — веб-приложение для ежедневного учёта личных финансов одного пользователя (или семьи через один аккаунт). Заменяет Excel-таблицу «Бюджет.xlsx» интерактивным интерфейсом с журналом операций, аналитикой и контролем лимитов по категориям.

Приложение **не является банковским агрегатором**: данные вводятся вручную или импортируются из Excel. Фокус — прозрачность расходов, несколько дебетовых карт, одна кредитная карта, месячные лимиты и накопления.

## Целевая аудитория

- Владелец продукта и его семья (личное использование)
- Пользователь, привыкший вести бюджет в Excel и хочет мобильный доступ через браузер/PWA
- Не требует финансовой грамотности на уровне бухгалтерии — UI на русском, быстрый ввод «за пару секунд»

## Ключевые задачи пользователя

1. **Быстро записать трату/доход** — с телефона, без лишних полей
2. **Видеть остатки на картах** — дебет + кредитка с долгом и доступным лимитом
3. **Контролировать категории** — «Еда», «Транспорт» и т.д. с месячными лимитами
4. **Находить проблемы** — операции без категории, без счёта, большой долг по кредитке
5. **Анализировать тренды** — сравнение с прошлым месяцем/годом, прогноз расходов
6. **Мигрировать с Excel** — одноразовый импорт истории

## Разделы приложения (навигация)

| Маршрут | Раздел | Назначение |

## Пользовательские сценарии (happy path)

### Новый пользователь

1. Регистрация → автоматическое создание seed-счетов и категорий
2. Мастер онбординга (если нет импорта и операций): счета → кредитка → категории с лимитами → первая операция
3. Ежедневный ввод через FAB «+» или блок «Быстрый ввод»

### Пользователь с Excel-историей

1. Настройки → Импорт Excel → загрузка `Бюджет.xlsx`
2. Python-скрипт парсит файл → PHP API записывает месяцы и транзакции
3. Онбординг **не показывается** (есть `importCompletedAt`)

### Ежедневное использование

1. Открыть главную → увидеть балансы и расходы месяца
2. Блок «Требует внимания» → перейти в журнал с фильтром (если есть проблемы)
3. Добавить операцию → категория подсказывается по названию
4. Раз в месяц — сверка кредитки с банком через модальное окно

## Что приложение сознательно НЕ делает

- Нет синхронизации с банками (Open Banking)
- Нет мультипользовательского доступа / семейных ролей
- Нет повторяющихся операций (recurring) — статус `planned` есть в модели, UI минимален
- Нет экспорта MySQL-бэкапа на проде (только SQLite локально)
- Нет push-уведомлений
- Нет офлайн-режима с синхронизацией (PWA manifest есть, service worker — нет)

## Конкурентный контекст

Продукт ближе к **YNAB / Money Lover / Zenmoney (ручной режим)**, но:

- Сильно упрощён под конкретную Excel-модель владельца
- Заточен под несколько дебетовых карт + одну кредитку
- Самохостинг на дешёвом shared-хостинге
- Кастомный визуальный стиль (Hello Kitty / dark fantasy)

## Метрики успеха (неформальные)

- Время добавления операции < 10 секунд с телефона
- Все операции имеют категорию и счёт
- Лимиты категорий отражают реальный бюджет
- Балансы карт совпадают с банком после сверки
